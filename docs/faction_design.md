# 设计文档：势力系统（Factions）

## 1. 目标
在现有“任务派遣/事件/主线章节”的基础上加入势力系统，让江湖更像江湖：
- 任务不再只是刷资源，而是会改变 **关系网与站队**
- 势力关系影响：任务池、奖励、风险、来访事件
- 为第4章「各派暗流」与第5章「武林大会」提供玩法基础

---

## 2. 势力清单（v1 建议 5 个）
> 先做少而精，后续可扩展到 8~10 个。

1) **正道盟**（主流门派联盟）
- 偏好：清/守
- 提供：大会资格、正道任务、名望加成

2) **魔教**（邪派联盟）
- 偏好：邪/烈/诡
- 提供：高收益高风险任务、邪道武学线索

3) **官府**（朝廷与地方衙门）
- 偏好：清（但更看“合法/秩序”）
- 提供：通缉/护送/清剿类任务；关系低会清剿你

4) **商会**（钱与物资渠道）
- 偏好：诡/守（交易稳定）
- 提供：折扣、拍卖会、稀有材料

5) **丐帮/情报网**（信息渠道）
- 偏好：速/诡
- 提供：情报任务、秘境线索、任务刷新增强

---

## 3. 核心变量

### 3.1 势力关系值（核心）
- 每个势力一条关系：`relation ∈ [-100, 100]`
- 初始建议：0 或按章节设定（例如官府-10、商会+10）

### 3.2 名声轴（你已有，势力会读）
- `alignment ∈ [-100, 100]`：正（+）↔ 邪（-）或正↔诡↔邪（可用区间）
- 势力对 alignment 有容忍区间：
  - 官府/正道盟对邪倾向更敏感
  - 魔教对正倾向更敏感

### 3.3 势力阈值事件（让数字有后果）
- 关系 ≥ 60：触发“盟会邀请/专属任务池/折扣/赠礼”
- 关系 ≤ -60：触发“围剿/断供/刺杀/通缉升级”

---

## 4. 势力如何影响游戏（四个入口）

### 4.1 影响任务池（最重要）
任务生成时：
- 每个任务模板可带 `factionId` 与 `weight`
- 关系值越高，该势力相关任务出现概率越大
- 关系值越低，该势力任务减少，但“敌对事件/围剿任务”增加

### 4.2 影响奖励与价格
- 商会：根据关系给商店折扣/溢价
- 正道盟：关系高时名望奖励加成
- 魔教：关系高时掉落邪道武学残页概率增加

### 4.3 影响事件池（门内来访/拜帖/纠纷）
- 正道盟关系高 → 来访拜帖增加
- 官府关系低 → 查税/通缉/搜山事件增加

### 4.4 影响主线与大会
- 第4章“参赛资格”：要求正道盟关系≥X 或 名望≥Y + 送礼/引荐事件
- 第5章“盟主票选”：势力关系决定支持票/暗箱操作选项

---

## 5. 数据结构（建议）

### 5.1 Faction 定义
```ts
interface Faction {
  id: string;                 // faction.righteous
  name: string;
  desc?: string;
  relation: number;           // runtime
  preferences: {
    labels: string[];         // 偏好标签，如 ['qing','shou']
    alignmentRange?: [number, number];
  };
  thresholds: {
    friendly: number;         // 60
    hostile: number;          // -60
  };
}
```

### 5.2 任务模板挂势力
```ts
interface MissionTemplate {
  id: string;
  type: string;
  factionId?: string;         // 可为空（中立任务）
  baseWeight: number;
  relationWeightCurve?: 'linear'|'step';
}
```

### 5.3 Effect 支持关系变化
- `faction_relation_delta { factionId, delta }`

---

## 6. 关系变化来源（v1规则）

### 6.1 任务结算
- 完成某势力任务：关系 +5~+15（按风险）
- 失败：关系 -3~ -10
- 任务中选择“灭口/栽赃”可能改关系与alignment

### 6.2 事件选项
- 拜帖/送礼：花银两换关系
- 拒绝/敷衍：关系下降

### 6.3 alignment 联动（被动变化）
- alignment 更邪 → 正道盟/官府关系每月轻微衰减（可选）
- alignment 更正 → 魔教关系衰减

---

## 7. v1 必做内容量（让系统跑起来）

### 7.1 势力数据
- 5 个势力定义（如上）

### 7.2 势力阈值事件（最少）
- 每势力：
  - 友好阈值事件 ×2
  - 敌对阈值事件 ×2
  - 中立交易/情报事件 ×2
- 总计：`5 × 6 = 30 条`（作为门内/来访事件子池）

### 7.3 任务模板挂钩
- 至少 12 个任务模板指定 factionId（覆盖 5 势力）

---

## 8. UI 需求（最小可用）
- 江湖/势力面板：
  - 关系条（-100~100）
  - 阈值提示（≥60、≤-60）
  - 本月关系变化来源（引用SettlementReport）
- 任务详情：显示该任务所属势力与可能影响

---

## 9. 验收标准
1. 关系变化可观察：完成/失败任务会改变关系。
2. 关系影响任务池：提高某势力关系后，该势力任务出现概率显著上升。
3. 阈值事件可触发：关系≥60 / ≤-60 会触发对应事件链。
4. 大会资格可被关系门槛控制（第4章落地）。

---

## 10. 迭代路线（v2/v3）
- v2：势力间敌对矩阵（正道盟 vs 魔教）影响你站队后果
- v3：动态江湖（势力强度随事件变化，地图节点易手）
